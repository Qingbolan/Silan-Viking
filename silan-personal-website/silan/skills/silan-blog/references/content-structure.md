# Content Directory Structure

Content lives under a `content/` directory inside a portfolio project (created by `silan init`). The hierarchy is **Collection → Item → File**. Each level carries a `.silan-cache` file holding sync configuration.

```
content/
├── .silan-cache              # Root content configuration
├── blog/
│   ├── .silan-cache          # Blog collection config
│   ├── post-name/
│   │   ├── .silan-cache      # Per-post config (multi-language / series)
│   │   ├── en.md             # English content
│   │   └── zh.md             # Chinese content
│   └── standalone-post.md    # Single-file post (no directory needed)
├── projects/
│   ├── .silan-cache
│   └── project-name/
│       ├── .silan-cache
│       ├── README.md         # Main project content
│       ├── QUICKSTART.md     # Quick start guide
│       ├── RELEASES.md       # Release notes
│       ├── DEPENDENCIES.md   # Dependencies
│       ├── Progress.md       # Progress documentation (+ Progress.zh.md)
│       ├── Reference.md      # Technical references (+ Reference.zh.md)
│       └── LICENSE
├── ideas/
│   ├── .silan-cache
│   └── idea-name/
│       ├── .silan-cache
│       ├── README.md         # Idea abstract / main content
│       ├── NOTES.md          # Progress notes
│       ├── REFERENCES.md     # References and resources
│       └── results.md        # Results and outcomes
├── episode/
│   ├── .silan-cache
│   └── series-name/
│       ├── .silan-cache      # Series overview (README with frontmatter)
│       ├── episode-01-setup.md
│       └── episode-02-content.md
├── resume/
│   ├── .silan-cache
│   ├── resume.md             # English resume
│   └── resume.zh.md          # Chinese resume
└── moment/                   # Short timeline updates
```

## Content types

| Type      | Purpose                                          | Key files                                  |
|-----------|--------------------------------------------------|---------------------------------------------|
| `blog`    | Articles, posts, vlogs, tutorial series          | `en.md` / `zh.md`, or single `*.md`         |
| `project` | Technical projects and portfolio items           | `README.md`, `QUICKSTART.md`, `RELEASES.md` |
| `idea`    | Research ideas and concepts                       | `README.md`, `NOTES.md`, `REFERENCES.md`, `results.md` |
| `episode` | Multi-part tutorial / video series                | series `README.md` + `episode-NN-*.md`     |
| `resume`  | Professional resume / CV                          | `resume.md`, `resume.zh.md`                 |

## Multi-language conventions

- **Filename variants**: `en.md` and `zh.md` in the same item directory, or `name.md` + `name.zh.md`.
- Create both at once with `silan new <type> <name> --language both`.
- Keep both language versions structurally consistent (same headings, same sections).

## `.silan-cache` files

These YAML/JSON files are generated by the CLI and carry per-collection / per-item sync metadata. The CLI maintains them — generally do not hand-edit unless debugging a sync issue.
